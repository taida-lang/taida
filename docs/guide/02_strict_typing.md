# 型のガチガチさ

## Taida は柔軟ではありません

他の言語は柔軟さを売りにすることが多いですが、Taida は**ガチガチな厳格さ**を売りにしています。

柔軟さは「何が起こるかわからない」の別名であり、暗黙変換は「予想外の挙動」の温床です。Taida の型システムは、開発者が考えなくていいように全てを決めています。デフォルト値があり、暗黙変換はなく、失敗は Lax で安全に処理されます。曖昧さはありません。

Taida の「ガチガチ」は、エラーで止めるのではなく、**Lax で安全に値を返す**ことでガチガチさと安全性を両立しています。

## 暗黙の型変換は存在しません

```taida reject
// 他の言語ならこう書けます。Taida ではコンパイルエラーになります。
"Count: " + 42        // コンパイルエラー: Str と Int の加算はできません
```

```taida
// これが正解です。明示的に変換してください。
stdout("Count: " + 42.toString())  // "Count: 42"
```

JavaScript は `"Count: " + 42` を `"Count: 42"` にしてくれます。便利ですが、`"5" - 3` は `2` になり、`"5" + 3` は `"53"` になります。便利の裏に予想外の挙動が潜んでいます。Python は `TypeError` を実行時に出します。正しい判断ですが、走らせるまでわかりません。

Taida は**コンパイル時**にエラーにします。走る前に止めます。

```taida
// 全て明示的に行います
num <= 42
str <= num.toString()           // "42"
Int["42"]() >=> back            // 42
ToFixed[num, 2]() >=> formatted // "42.00"
```

### Int と Float も暗黙には混ざらない

`Int` と `Float` は別の型として完全に独立しています。型チェッカーは `Int → Float` の暗黙昇格を一切行いません。算術 (`+`, `-`, `*`)、比較 (`<`, `>`, `>=`, `==`, `!=`)、関数引数、リストリテラル、条件分岐の各アーム、`<=` / `=>` の代入のすべてで、Int と Float が混在すると `[E1605]` などで reject されます。

```taida reject
// NG: 暗黙の Int→Float 昇格は存在しない
x <= 1 + 2.0           // コンパイルエラー: Int と Float の加算は禁止
y: Float <= 42         // コンパイルエラー: Int を Float の slot に渡せない
z: @[Float] <= @[1.0, 2, 3.0]  // コンパイルエラー: リスト要素の Int 混在
```

```taida
// OK: 明示変換モールドで揃える
Float[1]() >=> a           // Float[Int]() → Lax[Float] → unmold で Float
stdout(a + 2.0)            // 3.0

// OK: 真に Int で良いなら Int 同士に揃える
stdout(1 + 2)              // 3 : Int
```

明示変換に使うのは `Int[x]()` / `Float[x]()` の各モールドです (失敗時は Lax のデフォルト値 `0` / `0.0`)。比較や条件分岐でも同じルールが適用されるため、「片方だけ Float」「片方だけ Int」のような曖昧な書き方は許可されません。

```taida fragment
ratio: Float <= 0.5
threshold <= 100

// NG: 比較も型が揃っている必要がある
ratio > threshold           // コンパイルエラー

// OK: 明示的に同じ型に揃える
ratio > Float[threshold]() >=> _
```

## ゼロ除算は Lax で安全に処理します

Taida に `/` 演算子と `%` 演算子はありません。除算と剰余は `Div[x, y]()` と `Mod[x, y]()` モールドで行います。ゼロ除算は Lax を返し、プログラムを停止させません。

```taida fragment
Div[10, 3]() >=> result  // 3
Div[10, 0]() >=> result  // 0 (ゼロ除算: Lax のデフォルト値)
```

他の言語との比較:

| 言語 | `10 / 0` の結果 |
|------|-----------------|
| JavaScript | `Infinity` (壊れたまま走り続けます) |
| Python | 実行時エラー (`ZeroDivisionError`) |
| Java / Rust | 型によって挙動が分かれます (例外 / `Infinity` / パニック) |
| **Taida** | **Lax (デフォルト値)** — `has_value` で判別できます |

`Div[x, y]()` は Lax を返し、成功時は計算結果が、失敗時は型のデフォルト値がアンモールドで取り出されます。成功と失敗を判別したい場合は `has_value` を参照します:

```taida
lax <= Div[10, 0]()
lax.has_value  // false

lax2 <= Div[10, 3]()
lax2.has_value // true
```

剰余も同様です:

```taida fragment
Mod[10, 3]() >=> result   // 1
Mod[10, 0]() >=> result   // 0 (デフォルト値)
Mod[10, 0]().has_value     // false
```

## 範囲外アクセスは Lax で安全に処理します

リストの要素アクセスは `.get()` メソッドで行います。範囲外の場合は Lax を返し、プログラムを停止させません。

```taida fragment
items <= @[10, 20, 30]
items.get(1) >=> val    // 20
items.get(100) >=> val  // 0 (範囲外: デフォルト値)
```

他の言語との比較:

| 言語 | `array[100]` の結果 |
|------|---------------------|
| JavaScript | `undefined` (黙って壊れます) |
| Python / Java | 実行時エラー |
| Rust | パニック |
| **Taida** | **Lax (デフォルト値)** — `has_value` で判別できます |

`first()` / `last()` / `max()` / `min()` も同様に Lax を返します:

```taida fragment
@[1, 2, 3].first() >=> val  // 1
empty: @[Int] <= @[]
empty.first() >=> val        // 0 (空リスト: デフォルト値)

@[1, 2, 3].last() >=> val   // 3
empty.last() >=> val         // 0

@[1, 3, 2].max() >=> val    // 3
empty.max() >=> val          // 0
```

成功/失敗を判別したい場合:

```taida
empty: @[Int] <= @[]
lax <= empty.first()
lax.has_value  // false (空リスト)
```

## JSON はスキーマを通します

外部から入ってくる JSON データを Taida の型安全な世界にそのまま入れることはできません。JSON は**溶鉄**であり、スキーマ (鋳型) を通さなければ使えません。

```taida
Pilot = @(name: Str, age: Int, sync_rate: Int)

raw <= '{"name": "Shinji", "age": 14, "sync_rate": 78}'
JSON[raw, Pilot]() >=> pilot

pilot.name       // "Shinji"
pilot.age        // 14
pilot.sync_rate  // 78
```

スキーマなしの `JSON[raw]()` はコンパイルエラーです。`JSON` 値は不透明なプリミティブで、メソッドを持ちません。JSON の `null` やフィールド欠落はデフォルト値に変換され、内部に侵入することはありません。

詳細は [JSON 溶鉄](03_json.md) を参照してください。

## デフォルト値保証

> **PHILOSOPHY.md — I.** 深く考えずに適当にぶちこんでけ

「安全に処理する」と「デフォルト値がある」は一貫しています。

- `Div[10, 0]()` は Lax を返します — よくある失敗は Lax でデフォルト値にフォールバックします
- `Pilot(name <= "Rei")` で `age` は `0` — 省略されたフィールドにはデフォルト値が入ります
- `JSON[raw, Pilot]()` でフィールド欠損 → デフォルト値 — 外部データの不備はデフォルトで吸収します

よくある失敗には Lax で安全に処理し、省略された値にはデフォルトを与えます。全ての操作で「値が存在する」ことを保証するのが Taida の型システムの核心です。

| 型 | デフォルト値 |
|----|-------------|
| Int | `0` |
| Float | `0.0` |
| Str | `""` |
| Bool | `false` |
| @[T] | `@[]` |
| JSON | `{}` |

`null` はありません。`undefined` もありません。全ての変数には値があります。

## 型のガチガチさ = 安全 + 明示的

Taida の型のガチガチさは 2 つの意味を持っています。

### 1. 全てが明示的であること

```taida fragment
// 暗黙変換はありません
"Count: " + 42                 // コンパイルエラー
"Count: " + 42.toString()      // OK: 明示的な変換

// 操作はモールドで明示的に行います
Upper["hello"]()               // "HELLO"
Div[10, 3]()                   // 除算もモールドで

// JSON はスキーマ必須です
JSON[raw, Pilot]()             // OK: スキーマ指定
JSON[raw]()                    // NG: スキーマなし
```

### 2. 全ての操作が安全であること

```taida
// ゼロ除算 → Lax で安全に
Div[10, 0]() >=> result        // 0 (プログラムは止まりません)

// 範囲外アクセス → Lax で安全に
@[1, 2].get(100) >=> val       // 0

// 型変換失敗 → Lax で安全に
Int["abc"]() >=> num           // 0

// JSON のフィールド欠損 → デフォルト値
JSON['{}', Pilot]() >=> pilot  // 全フィールドがデフォルト値
```

### 型チェッカーが守るもの

```taida reject
// 存在しないフィールドへのアクセス
pilot <= @(name <= "Asuka")
call_sign <= pilot.call_sign  // コンパイルエラー: フィールド 'call_sign' は存在しません

// 型不一致
add x: Int y: Int = x + y => :Int
result <= add("hello", 5)     // コンパイルエラー: 第1引数で Str 型を Int 型に適用できません

// 単一方向制約
data => Filter[_, _ x = x > 0]() <= result  // コンパイルエラー: 単一方向制約違反
```

## ガチガチの何がいいのか

1. **バグが実行前にわかります** — コンパイル時に型エラーを検出します
2. **よくある失敗で止まりません** — Lax がデフォルト値で安全に処理します
3. **AI が安全なコードを生成できます** — 暗黙の挙動がないため、AI の出力が予測可能です
4. **人間は眺めるだけで構いません** — 型が合っていれば、コードの構造が正しいことが保証されます

ガチガチだからこそ、安心してぶちこめるのです。

## まとめ

| 原則 | Taida の対応 |
|------|-------------|
| 暗黙変換 | 存在しません。コンパイルエラーです |
| ゼロ除算 | `Div[x, y]()` が Lax を返します。`has_value` で判別できます |
| 範囲外アクセス | `.get()` が Lax を返します。`has_value` で判別できます |
| 型変換 | モールドで明示的に。`Int[x]()`, `Str[x]()` 等。全て Lax を返します |
| JSON | スキーマ必須 (`JSON[raw, Schema]()`)。メソッドなし。詳細は [03_json.md](03_json.md) |
| null | 存在しません。全ての型にデフォルト値があります |
